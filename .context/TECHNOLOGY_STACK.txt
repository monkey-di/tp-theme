TECHNOLOGY STACK - TEBE POVERYAT THEME
=======================================

PROJECT TYPE
============
WordPress Custom Theme (Classic PHP)
Not a full block theme - uses template-parts for component composition

FRONTEND TECHNOLOGIES
=====================

CSS Framework
  Tailwind CSS v4.0.0-alpha.14
  - Alpha/Beta version (potential breaking changes)
  - Configuration via @theme directive in src/input.css
  - NOT traditional Tailwind v3 (no tailwind.config.js)
  - BEM classes combined with Tailwind utilities

Fonts
  Geologica (Google Fonts)
    - Used for: Body text, descriptive content
    - Weight: 100-900
    - Loaded: Google Fonts API
  
  Akrobat (Local)
    - Used for: Headers (h3-h6), UI elements, buttons
    - Available: Regular, Bold, ExtraBold, Black
    - Weights: 400, 700, 800, 900
    - Format: OTF, TTF, woff2
  
  Ura Bum Bum SP (Local)
    - Used for: Large display text (h1, h2)
    - Used for: Decorative/accent text
    - Format: OTF

JavaScript Libraries
  Swiper.js v11.0.0
    - Carousel/slider library
    - Used for: 6 different sliders (Hero, Friends, Media, Materials, Histories, Team)
    - Features: Loop, autoplay, navigation arrows, progress bar, breakpoints
    - Source: npm, copied to assets/vendor/swiper/ during build
  
  Vanilla JavaScript
    - Mobile menu toggle (main.js)
    - Slider initialization (sliders.js)
    - DOM manipulation, event listeners
    - NO frameworks (Vue, React, Angular)

BACKEND TECHNOLOGIES
====================

WordPress
  Version: 6.x+
  Theme Type: Classic PHP (template-parts based)
  NOT: Full block theme (block.json)
  NOT: FSE/Full Site Editing theme
  
PHP
  Version: 7.4+
  Template System: get_template_part() with arguments array
  Enqueue System: wp_enqueue_script(), wp_enqueue_style()
  Features Used:
    - Theme support (title-tag, post-thumbnails, custom-logo)
    - Template hierarchy
    - Actions/Filters (wp_enqueue_scripts, after_setup_theme)
  
No Plugin Dependencies
  - All functionality in theme files
  - No custom post types
  - No advanced custom fields
  - No WooCommerce
  - No complex WordPress features

BUILD & DEVELOPMENT TOOLS
==========================

Package Manager: npm
  - Node.js v14+ required
  - package.json with 3 dependencies

Build Tool: Tailwind CLI
  - Command: npx @tailwindcss/cli
  - Input: src/input.css
  - Output: assets/css/output.css
  - Modes: Watch (--watch) and Build (--minify)
  
CLI Tasks:
  npm run dev          # Watch + recompile
  npm run build        # Production build (minified)
  npm run copy-assets  # Copy Swiper from node_modules

Version Control: Git
  - Repository: https://github.com/monkey-di/tp-theme
  - Branch: master
  - Commits tracked in git history

Docker (Optional)
  docker-compose.yml
  Services:
    - WordPress (latest) on port 8080
    - MySQL 5.7
  Volumes: Theme mounted for hot reload

ARCHITECTURE DECISIONS
======================

Template-Parts Composition
  - NOT monolithic templates
  - Separate component files in template-parts/
  - Front-page.php orchestrates sections
  - get_template_part() pattern for reusability

CSS Architecture
  - Mobile-first responsive design
  - Tailwind utilities + BEM naming
  - Single CSS file (output.css)
  - Color system: @theme configuration

JavaScript Strategy
  - Vanilla JS (no build processing)
  - Swiper.js for carousels (external library)
  - Minimal JS footprint (menu + slider init)
  - NO frontend build (TypeScript, webpack, etc)

PHP Philosophy
  - Classic WordPress theme structure
  - No custom WP-CLI commands
  - No advanced PHP frameworks
  - Simple, readable code
