Ты работаешь над WordPress темой "Tebe Poveryat" (Hybrid Theme: Classic PHP + Tailwind CSS v4).

**Текущее состояние:**
- Репозиторий: https://github.com/monkey-di/tp-theme
- Главная страница (`front-page.php`) сверстана **Mobile First** и разбита на `template-parts/home/*.php`.
- **UI Kit** внедрен (`template-parts/components/`).
- **Слайдеры (Swiper.js)** подключены и работают (Hero, Friends, Media, Materials, Histories, Team). Секция Projects — статичная.
- Сборка: `npm run dev` (Tailwind CLI).

**Твоя задача на эту сессию:**
Приступить к **Адаптиву под Desktop**.
1. Проанализируй текущую мобильную верстку в `template-parts/home/`.
2. Добавь классы Tailwind (`md:`, `lg:`, `xl:`) для корректного отображения на планшетах и десктопах в соответствии с макетом пользователя.
3. Следи за тем, чтобы слайдеры (Swiper) корректно вели себя на десктопе.
4. Не ломай обратную совместимость, никогда и ни при каких обстоятельствах уже существующая мобильная вёрстка не должна пострадать
5. Старайся хранить вёрстку отдельных компонентов в отдельных css-файлах(в том числе можно рефакторить мобильные utility-классы, с полным сохранением обратной совместимости)

Важно учитывать, что пользователь предоставляет макет, а не готовую итоговую вёрстку. Он даёт общее представление о том, как элементы должны выглядеть, однако требует полноценной доработки.
